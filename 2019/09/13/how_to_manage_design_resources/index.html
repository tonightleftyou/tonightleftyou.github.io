为什么要管理设计资料

因为是刚需。

  • 绘制图像从互联网上下载下来的笔刷纹理；
  • 日积月累的免费优质高清图像；
  • 从各种地方收集的灵感图像；
  • …

以及从更重要的工作流程上考虑：

  • 项目稿件的版本管理；
  • 资料的安全容灾策略；

因此，本篇文章将从流程来说明如何管理设计资料。

项目初期启动阶段：灵感收集/风格面板

相信大多数项目的启动初期，设计师都有一个和各方面沟通的过程，虽然大多数时候避免不了「大气，高端，上档次」这样的让人摸后脑勺的需求。但是多少是会有一个大概的概念的：
一个描述项目初期印象的导图

从导图中可以理出一个思路：

  • 寻找整理同类产品的界面参考
  • 尽量收集符合感觉气质的各种图片
  • 准备各种设计阶段需要用到的素材，比如笔刷/图标/纹理

那么这里的工具需求是：
如何存储这些图片，方便在项目的任何阶段都可以快速的翻阅？

  1. 本地文件夹：这是一个传统的方法，需要分门别类的命名查找，最大的问题是比较繁琐，每一张都需要保存，命名，日积月累占据硬盘空间。
  2. pin/花瓣 等工具：这是很多设计师都在采用的办法，这样的图片站点本身就是大量的灵感库，可以很快速的从别人的整理中得到启发。唯一的坏处是：如果网站被墙或者下线，你的东西就等于消失了。——pin目前在国内是不可访问状态，花瓣网也曾经出现过一个月无法访问的情况。
  3. eagle：这是一款本地软件，采用了花瓣+本地文件树管理的结构，同时支持大量的批操作——可以说完美的解决了初期的设计资料管理需求，相对于上面两个方法，可以算得上的缺点仅仅只有需要付费，以及占据空间这两点。

目前我采用的是第二种和第三种办法并行，第二种作为粗略整理，第三种作为最后的整理。

项目进行阶段：稿件的版本控制

一旦设计工作开启，不可避免的是版本控制问题，这一方面，市面上有好几种解决策略。

  1. abstract 为首的基于git的版本控制软件，这一类基本要求付费，优势是确实严格按照产品设计的原理来打造软件。——某种程度说是团队项目进度更新管理工具也是可以的。

    对于已经在使用其他团队项目管理工具（比如jira/tpad/zendao/redmine/teambition）的团队有一定的迁移成本。——在真实的项目开发环境中，很少有软件能够接管一支队伍从头到尾所有需求的。

    如果并不涉及多个设计师协同工作或者设计作为整个开发中特别强调的一环，这样的工具有一点杀鸡用牛刀的感觉。

  2. 群晖/坚果云/百度云 等网盘性质的团队套餐，这些操作上更简单，功能上也更强调配合已经有的流程——对于一些中型大型设计团队，是典型的版本控制选择。这样的套餐一般需要付费，从体验上，是非常优秀的仓库，并不具备太多的协作功能。最主要的是，他们主要解决的是设计师之间的版本管理问题。

  3. 本地文件夹——这是一种非常常见的策略，设计师需要自己管理好，交接给别人的时候也是打包。在设计师人数非常少，协作需求并不多的时候，足够使用，坏处是：依赖设计师的良好习惯，协作依赖其他工具的交流，以及多个版本的冗余比较占据空间。

  4. 蓝湖——基于设计稿的对接平台，对团队来说，仅仅看到的是已经确认的版本。而对设计师来说，同一页面的所有历史版本都可以切换查看。另外平台也提供了产品经理的原型各个页面的各个版本。以及一些评论批注提醒功能。——功能上没有全面引入git管理，面向的对象也是主要以产品和设计两个角度的对接来考虑。

笔者一开始采用的是方法3，但是随着团队协作人员的增加以及多个需求并行的情况越发的多起来（特别是多数情况都是远程协作），发现引入一个可以恰好不打乱原本的流程，而又可以方便团队的工具势在必行。经过多方比较：

  • 给前端开发人员提供便捷的自动标注；（像素大师/sketch measure / 墨刀 / 蓝湖 / xiaopiu 均可以）
  • 和产品经理能够便捷减负的对接稿件更新；（墨刀/蓝湖/xiaopiu）
  • 便于设计师查看历史版本；（蓝湖）

于是接入了蓝湖，唯一需要担心的是这一套东西可能本地部署更安全一些。

项目的安全措施：云端同步备份

你永远也不知道什么时候会失去你的电脑。——by 鲁迅没这么说过

  • 即便是mac 自带的time machine ，精度也仅仅是一个小时。——还要祈祷空间足够大，能够塞下那么多的备份。

  • 如果遭遇了“电脑损坏需要维修至少等待一个星期+稿件两天后需要提交”这样的至暗时刻。——跳楼吧，还可以享受下蹦极的快乐。

比较了市面上的几款网盘应用，结合我国的特殊网络情况（dropbox啥的不好用），以及个人钱包的厚度（群晖之光照耀我心）。

我选择了坚果云作为资料的备份方案。

并不需要打开什么网页软件拖拽/选中 上传。仅仅需要将文件夹路径录入，之后付费日子里的文件夹下的变动，以分钟级别的速度自动同步到云端。并自动记录每个变动的历史版本（最多可以查看同一文件的最近100个版本）

将eagle的仓库路径，工作文档路径全部录入。——即便发生车祸电脑碎成了渣渣：

  1. 立马买/借/租 一台mac；
  2. 安装好sketch ，eagle ， 坚果云，并登陆对应账号；
  3. 一键同步需要的内容；
  4. 愉快的开始一天的工作吧。

    总结

  • 付费不是缺点，免费的可能是最贵的。
  • 安全备份不做，对不起队友对不起自己。
